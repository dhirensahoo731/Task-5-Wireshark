Wireshark was used to capture network traffic on the Wi-Fi interface for approximately one minute. During the capture, web browsing activity was performed to generate network packets. The captured traffic was filtered and analyzed using protocol filters.
Three protocols were identified:
DNS (Domain Name System) – Used to resolve domain names into IP addresses.
TCP (Transmission Control Protocol) – Used for reliable communication and data transfer between devices.
HTTP (Hypertext Transfer Protocol) – Used for web page requests and responses.
The packet analysis showed DNS queries and responses, TCP connection establishment and data transfer, and HTTP GET requests with corresponding server responses. The capture was successfully saved as a .pcapng file for future analysis.
Conclusion:
This activity provided hands-on experience with packet capture and protocol analysis using Wireshark, improving understanding of how network communication occurs over DNS, TCP, and HTTP protocols.
